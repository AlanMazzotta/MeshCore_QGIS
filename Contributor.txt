Hey, I'm Alan — new MeshCore user in Portland, studying GIS. I posted a terrain viewshed project to the local channel recently and someone pointed me toward MeshMapper, which I hadn't seen before.

I noticed the PNW coverage data follows roads pretty closely, which makes sense for wardriving. I built an open-source pipeline that runs terrain line-of-sight viewshed analysis against the public MeshCore API data — it identifies areas that are geometrically reachable from existing nodes but off-road, so they wouldn't typically get tested by wardriving: trail corridors, forest service roads, hillsides. I used Portland as the example dataset.

My thought is I could use that to prioritize where to go war-walk and hike with MeshMapper, focusing on off-road terrain rather than streets — to fill in areas that appear dark just because nobody's been there with a radio.

I don't know how contributions work but I'd love to connect if you're open to it. The project is at https://github.com/AlanMazzotta/MeshCore_QGIS if context is useful.
